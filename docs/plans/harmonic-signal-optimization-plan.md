# 谐波形态检测与交易信号优化方案 v4

> 本版在 v3 基础上，吸收 harmonious-trading-advisor（三支柱融合引擎）的核心方法论，
> 新增**信号有效性验证**与**统计检验**两大支柱，目标只有一个：提高胜率信号发现的有效性。
> 直接替代 v3。
>
> 参考项目可移植的 7 个关键设计：
> 1. **只用最近 4–5 个枢轴点做形态识别**——天然免疫陈旧形态（我们事故的根治思路）；
> 2. **多窗口形态稳定性检验**——同一形态在全窗口/去头/去尾三个窗口重复检测，仅单窗口出现的形态疑似人造信号，直接否决；
> 3. **统计检验闸门**——估算夏普比率 + 波动率目标仓位，不达标不放行；
> 4. **量化陷阱识别**——假突破率、止损猎杀率、量能高潮率、PRZ 支撑失败校验；
> 5. **量价真实性**——量价对齐率 + 脉冲率 + 量能自相关；
> 6. **连续型比率匹配评分**——`max(0, 1−dev×3.5)` 替代二元容差，形态质量分更细腻；
> 7. **校正因子反馈环**——用近期历史预测的准确率/偏差自动校正后续得分（±30 分）。

---

## 0. 已确诊事故（SOXLUSDT 案例，v3 已列，保留）

2026-07-17 实测：SOXLUSDT 现价 126.23，系统输出"做空 / 入场区 191.04 / C 级"。
根因：陈旧形态未过滤（距现价 45×ATR）、状态机缺失效态、双口径输出矛盾、止损方向无校验、ATR 失真。
修复方案见第 2 章 P0，**v4 不变**。

## 1. 总体架构：四支柱信号有效性引擎（v4 核心变化）

```
                    ┌─────────────────────────────────────────┐
行情数据 → 形态检测 → │ P1 谐波检测   P2 共振评分                │
                    │ P3 统计检验   P4 信号有效性验证（新增）    │
                    └──────────────┬──────────────────────────┘
                                   ▼
                    动态权重融合（regime 感知）→ 候选过滤器 → 信号引擎
                                   ▼
                    出场计划 / 仓位 / 冷却 / 理由 → API / 前端
                                   ▼
                    回测 + 校正因子反馈环（自我校准）
```

**动态权重融合（移植反量化 regime 概念）**：

| 市场状态 | P1 谐波 | P2 共振 | P3 统计 | P4 有效性 | 处置 |
|----------|---------|---------|---------|-----------|------|
| 正常（normal） | 40% | 30% | 15% | 15% | 标准 |
| 高操纵（high_quant） | 20% | 25% | 15% | **40%** | 止损放宽至 3.0×ATR、仓位 ×0.6、A 级门槛 +10 分 |

regime 由第 4.1 节的量化冲击分判定（≥60 high_quant / 35–59 moderate / <35 normal）。

## 2. P0：事故根因修复（继承 v3，不变）

1. **候选有效性过滤器**：空间距离 ≤3×ATR、D 点时间新鲜度（≤max(20 根, 1.5×CD 腿)）、退化 PRZ、失效检查、已走完检查，任一不过即丢弃；
2. **方向不变量断言**：多 `stop < entry ≤ TP1 < TP2 < TP3`，空镜像，违反即丢弃；
3. **统一输出口径**：`technical_result` 旧字段一律从选中 Signal 派生，无信号则为 null；
4. **ATR 稳健化**：`min(全窗口ATR, 近100根ATR)`。

**v4 增补**：过滤器再吸收参考项目"最近枢轴"思想——`Candidate` 增加 D 点距当前 K 线数（`d_age_bars`），>20 根直接 stale，与价格距离校验双保险。

## 3. P1：形态检测增强

### 3.1 连续型比率匹配评分（替代二元容差）

参考项目 `ratioMatch = max(0, 1 − dev/ideal × 3.5)`：

- 每个腿的实测 fib 比率与教科书比率（Gartley: B=61.8%XA、D=78.6%XA；Bat: D=88.6%；Butterfly: D=127.2–161.8%；Crab: D=161.8%）计算连续偏差分；
- 加权合成形态质量分（0–100）：B 点 35% + D 点 30% + C 点 20% + CD 延伸 15%（XABCD）；AB=CD：腿长 40% + BC 回撤 35% + CD 延伸 25%；
- 质量分 < 40 的候选直接丢弃（参考项目 confidence > 30 才采纳的同构门槛）。

### 3.2 枢轴新鲜度（吸收"最近 4–5 枢轴"思想）

- 形态五点中，D 点必须是全部枢轴中**最近 3 个之一**；否则判 stale；
- 与 v3 的时间/空间过滤并列，三保险。

## 4. P4：信号有效性验证（v4 新增支柱，胜率提升主战场）

四个纯函数验证器，产出 `authenticity_score (0–100)` 与否决标志。**任何否决标志命中 → 信号直接丢弃，不走评分。**

### 4.1 量化冲击/市场操纵检测（regime 判定）

对最近 60–100 根 K 线计算：

| 指标 | 含义 | 权重 |
|------|------|------|
| 跳空频率 | `|open − prevClose|/prevClose > 0.8%` 的占比 | 25% |
| 日内反转率 | 振幅 >3% 但实体 <0.5% 的 K 线占比（插针特征） | 25% |
| 量能变异系数 | `std(volume)/mean(volume)` | 30% |
| 肥尾事件率 | 收益偏离均值 >3σ 的占比 | 20% |

合成 quant_score → regime（normal/moderate/high_quant），驱动第 1 章的动态权重与风控放宽。

### 4.2 多窗口形态稳定性（参考项目最精妙的设计，直接移植）

- 同一数据跑三次检测：全窗口、去掉最近 5 根、去掉最早 5 根；
- **判定**：三窗口形态一致 → 85 分；两窗口一致 → 55 分；类型不一致 → 25 分；**形态仅在全窗口出现 → 20 分且标记"疑似人造"**；
- 得分波动 <10 分 +10，>30 分 −20；
- 移植到我们的实现：对 `df[:-5]` 与 `df[5:]` 各跑一次 `detect_patterns`（成本 ×3，可接受；仅对 A/B 级候选执行以省耗时）。

### 4.3 量化陷阱风险（否决项）

| 陷阱 | 判定 | 处置 |
|------|------|------|
| 假突破频繁 | 近 N 根中"刺破前 5 根高/低点 >1% 又收回"占比 >15% | −20 分；>25% **否决** |
| 止损猎杀 | "创 3 根新低后下一根收复高点"占比 >10% | −15 分 |
| 量能高潮 | "量增 2 倍但实体 <1.5%"占比 >12% | −15 分 |
| PRZ 支撑失败 | K 线曾进入 PRZ 区间但收盘跌破 PRZ 下沿（多头） | **否决** |

### 4.4 量价真实性

- 量价对齐率（涨跌与量增同向的占比）40% + 脉冲率（量 >2.5× 均值占比，反向计分）40% + 量能自相关 20%；
- <40 分：价格行为因子减半；<25 分：否决。

## 5. P3：统计检验闸门（v4 新增支柱）

### 5.1 估算夏普闸门

- 近 20 根 K 线日收益估算年化夏普（无风险利率 2.5%）；
- **闸门：Sharpe > 0.1 才允许 A/B 级**；否则最高 C 级（观察）。

### 5.2 波动率目标仓位

- `ATR% = ATR(14)/close`；目标波动 2.5%；
- `仓位乘数 = clamp(2.5% / ATR%, 0.5, 1.5)`——高波动标的自动缩仓，低波动放宽；
- 与 v3 的风险预算公式串联：`最终仓位 = 风险预算 × 仓位乘数 / (止损距离 + 费用)`。

### 5.3 相关性敞口（保留 v2）

以 BTC 为"指数"：Pearson 相关 >0.8 的同向信号合并敞口 ≤2%。

## 6. 入场、出场、冷却、仓位（继承 v3，压缩）

- **入场**：默认确认入场（收回 PRZ + 反转 K 线 + 放量），限价模式挂 PRZ 61.8% 深处；
- **门槛**：EMA200 同周期 + HTF EMA21/55 双层趋势、ADX>20（A/B 级硬门槛）；
- **分级**：A ≥75 且净 RR(TP2)≥2.0 且全部闸门通过；B 60–74；C 45–59 仅观察；<45 丢弃；**v4 增补：P4 否决项命中即丢弃，与分数无关**；
- **出场**：Phase1 结构止损 → TP1 平 50% 移保本 → TP2 平 30% → Chandelier 追踪（highest(22)∓2×ATR）→ TP3 清仓；时间止损 1.5×CD 腿；
- **冷却**：同 symbol 同向 4h；全局 active ≤5；日内 −3R 熔断；
- **仓位**：风险预算 0.5%/1% × 波动率乘数，反推数量；杠杆提示满足强平缓冲 ≥3× 止损距离。

## 7. 校正因子反馈环（v4 新增，自我校准）

移植参考项目的 correction factor loop，对接 P3 回测：

1. 滚动回测最近 N 次分析（5 日 cutoff 法：用 T−5 之前的数据跑完整管线，对比 T−5→T 的实际走势）；
2. 计算方向准确率 accuracy 与系统性偏差 bias；
3. `correction = (0.5 − accuracy) × 0.4 + clamp(bias/8, −1, 1) × 0.15`，限幅 ±0.3；
4. 后续评分 `score += correction × 100`（±30 分），自动惩罚系统性高估、奖励系统性低估；
5. 落库 `signal_outcomes`，准确率 <40% 时自动告警并冻结 A 级产出。

---

## 8. 回测与校准（继承 v2/v3，增补）

1. 逐时点回放（`df[:t]` 切片重跑全管线，含 P4 验证器）+ 前视偏差审计测试（CI 红线）；
2. **5 日 cutoff 快速验证**（移植）：每次部署前对 20 个主流币跑一次轻量验证，出准确率/偏差快报；
3. 分桶统计净期望值/胜率/PF/回撤（形态 × 市场 × 周期 × 级别，样本 <100 标 insufficient_data）；
4. walk-forward + isotonic 校准分级阈值；冷却期与两阶段出场必须进回放；
5. 费用敏感性三档 + 牛熊震荡分段报告。

## 11. 设计原则校验

- **KISS**：P4 四个验证器都是"K 线数组进 → 分数/否决出"的纯函数；regime 只有两档权重表；校正因子一个公式。
- **高内聚低耦合**：验证逻辑全部收在 `domain/validation.py`（新）；信号引擎只消费总分与否决标志；参考项目思想全部转写为我们的数据结构，不引外部依赖。
- **有效性优先于覆盖率**：P4 否决项优先于一切评分——宁可漏掉真信号，不可放过假信号，这是"提高胜率"的根本取舍。

## 12. 风险与对策

- **多窗口检测成本 ×3**：仅对通过初筛的候选执行，常态开销可控；
- **否决项误伤**：所有否决阈值先进 `calibration.json`，由 P3 回测数据校准，不拍脑袋；
- **校正因子震荡**：限幅 ±0.3 + 准确率 <40% 才冻结，避免反馈环自激；
- **不承诺胜率**：交付严格风控 + 可验证正期望，页面保留免责声明。
